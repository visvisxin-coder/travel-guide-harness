# Quality Rules

## Scoring Components

| Component | Weight |
|---|---:|
| Destination research | 15% |
| Social trend usefulness | 10% |
| Route feasibility | 20% |
| Transport convenience | 15% |
| Guide usefulness | 15% |
| Visual design | 15% |
| Verification readiness | 15% |

For return-visit guides, include repeat-visit fit inside route feasibility and guide usefulness. A route that repeats excluded or already visited places should not pass client-ready review.

If transport planning is in scope, renormalize weights or use transport convenience as a required component. A guide with vague transport advice should not pass client-ready review.

If social trends are in scope, score them by usefulness and filtering quality, not by how many trending items are included.

## Gates

| Gate | Score |
|---|---:|
| Draft | 70 |
| Preview | 80 |
| Client-ready | 90 |
| Publication | 95 |

## Severity

- Discovery: encouraging, catch missing coverage.
- Route planning: strict about geography and pacing.
- Writing: strict about usefulness and factual consistency.
- Design: strict about readability and overflow.
- Final verification: pass/fail for output readiness.
