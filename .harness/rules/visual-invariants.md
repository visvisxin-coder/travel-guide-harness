# Visual Invariants

These are non-negotiable rules for PDF and image output.

**VINV-1.** Text must not overflow its container.

**VINV-2.** The polished HTML version is the primary visual artifact.

**VINV-3.** PDF pages must be readable when printed on A4.

**VINV-4.** Mobile image output must be readable at 1080px width.

**VINV-5.** Each visual output must have a clear hierarchy: title, overview, daily routes, transport, backup plans.

**VINV-6.** Cards may be used for repeated itinerary items, but page sections should not become nested cards.

**VINV-7.** Color contrast must remain readable.

**VINV-8.** Avoid decorative clutter that competes with route information.

**VINV-9.** The city and trip length must be visible in the first viewport or cover.

**VINV-10.** Metro map modules must remain readable in HTML, image, and PDF outputs.
